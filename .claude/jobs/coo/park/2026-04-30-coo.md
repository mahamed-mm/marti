## Gotchas for next session

- **Subagent harness can hijack delegation into plan-approval gates.**
  When you see "Plan written to /Users/moha/.claude/plans/..." in an
  Agent tool result, the agent didn't execute — it's waiting for plan
  approval. Two paths: (a) use `SendMessage` with a `plan_approval_response`
  protocol message, OR (b) execute in the main session if the work is
  small enough. The `mode: "acceptEdits"` Agent parameter does NOT reliably
  bypass this. Today this cost ~10 minutes before pivoting to do the work
  directly.

- **`-only-testing:Bundle/Suite/method` does NOT work for Swift Testing
  per-method filters.** It silently runs zero tests and reports
  `** TEST SUCCEEDED **` because nothing failed. Filter at suite level
  (`-only-testing:Bundle/Suite`) and read per-`Test case` lines, or use
  `xcrun xcresulttool get test-results tests --path <bundle>` for
  structured results. Documented in ios-engineer's park doc too.

- **Test count of 99/99 is the new anchor.** Both prior baselines (98/98
  and 97/97) had off-by-one errors. Trust the `xcodebuild` per-case lines,
  not historical doc claims.

- **Listing → seed → refresh pattern needs class-of-bug vigilance.** When
  `listing` is reassigned by `refresh()`, every ViewModel property derived
  from `listing`'s shape (array sizes, ID presence, conditional flags) is
  at risk of going stale. Today's fix patched one instance; audit the
  next detail-style ViewModel that gets a similar shape with this lens.
